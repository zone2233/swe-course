# Roadmap

## Project

Console ToDo App

## Overview

This roadmap describes how the console ToDo app was developed in small iterations.

---

## Iteration 1: Basic Menu and Task Creation

### Goal

Create the basic console menu and allow the user to add and view tasks.

### Features

- Show console menu
- Add a task
- Reject empty task titles
- Show all tasks
- Show a message when no tasks exist
- Exit the program

### Related Requirements

- REQ-001: Show menu
- REQ-002: Add task
- REQ-003: Reject empty task title
- REQ-004: Show tasks
- REQ-005: Show message when no tasks exist
- REQ-009: Exit program

### Expected Outcome

At the end of iteration 1, the user can start the program, add tasks, view tasks, and exit the program.

---

## Iteration 2: Task Completion, Deletion, and Error Handling

### Goal

Add the ability to update and remove tasks, and make the program display an error message against invalid input.

### Features

- Mark a task as done
- Delete a task
- Handle invalid task numbers
- Store tasks during the current program session

### Related Requirements

- REQ-006: Mark task as done
- REQ-007: Delete task
- REQ-008: Handle invalid task numbers
- REQ-010: Session-only storage

### Expected Outcome

At the end of iteration 2, the app behaves like a complete simple todo manager for one program session.

---

## Possible Future Iteration 3: Persistence and Improvements

### Goal

Improve the app by adding features not included in the current version.

### Possible Future Features

- Save tasks to a file
- Load tasks when the program starts
- Add task deadlines
- Add task categories
- Edit existing tasks
- Add automated tests

### Related Future Requirements

These features are not implemented in the current version, but could be added later.
